# ERP 系统架构概览

## 1. 系统定位

ERP 系统是一个企业资源管理系统，主要面向数据中心资产管理场景，包括：
- IDC（数据中心）管理
- 服务器资产管理
- 网络资源管理
- 电源与能耗管理
- 库存与采购管理

## 2. 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                    erp (Root POM)                            │
├─────────────────────────────────────────────────────────────┤
│  platform-common          │  platform-common-db              │
│  (公共工具、常量、异常)    │  (JPA公共访问层、公共实体)       │
├───────────────────────────┴──────────────────────────────────┤
│  业务子模块 API（接口定义，无实现）                           │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐         │
│  │ user-api     │ │ idc-api      │ │ server-api   │  ...    │
│  └──────────────┘ └──────────────┘ └──────────────┘         │
├──────────────────────────────────────────────────────────────┤
│  业务子模块 SERVICE（具体实现）                               │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐         │
│  │ user-service │ │ idc-service  │ │server-service│  ...    │
│  └──────────────┘ └──────────────┘ └──────────────┘         │
├──────────────────────────────────────────────────────────────┤
│  platform-app (聚合启动器，全局 SpringBoot 入口)              │
└──────────────────────────────────────────────────────────────┘
```

## 3. 模块依赖规则

- `*-service` 模块只能依赖 `*-api` 模块，禁止依赖其他 `*-service` 实现
- 所有跨模块调用通过 `*-api` 中定义的 Service 接口
- `platform-common-db` 是唯一允许的 JPA 基础设施层

## 4. 数据库架构

每个模块使用独立的 SQLite 数据库文件：

| 模块 | SQLite 文件路径 |
|------|----------------|
| platform-common-db | `./data/erp.db` |
| user-service | `./data/user.db` |
| audit-service | `./data/audit.db` |
| idc-service | `./data/idc.db` |
| server-service | `./data/server.db` |
| network-service | `./data/network.db` |
| power-service | `./data/power.db` |
| inventory-service | `./data/inventory.db` |

## 5. 部署模式

- **全局单进程**: `platform-app` 聚合所有模块，端口 8080
- **独立微服务**: 各模块可独立启动，端口 8081-8085
