## 1. 系统概述和目标

### 1.1 项目背景

随着云计算产业的快速发展，数据中心基础设施规模持续扩大，传统的手工管理方式已无法满足现代云服务商对基础设施资源的高效管理需求。云计算数字供应链ERP系统旨在为云服务商提供一套完整的、数字化的基础设施资源管理解决方案，实现从机房规划、设备采购、资源分配到运维管理的全生命周期管理。

### 1.2 系统定位

本系统定位为云计算基础设施领域的专业ERP系统，聚焦于：

- **IDC机房管理**: 数据中心机房的全生命周期管理
- **服务器资产管理**: 服务器设备的采购、部署、运维、报废全流程
- **网络资源管理**: 网络设备、IP地址、带宽资源的管理
- **电力能源管理**: 电力容量、能耗监控、成本核算
- **库存采购管理**: 备件库存、采购流程、供应商管理

### 1.3 核心目标

| 目标维度 | 具体指标 |
|---------|---------|
| **资产可视化** | 实现100%资产的可视化追踪，资产位置、状态实时可查 |
| **运维效率** | 资产盘点效率提升300%，故障响应时间缩短50% |
| **成本优化** | 通过精准的资源规划，降低10-15%的闲置资源成本 |
| **合规管理** | 满足等保2.0、ISO27001等合规要求 |
| **决策支持** | 提供实时数据支撑，辅助容量规划和采购决策 |

### 1.4 目标用户

| 用户角色 | 主要职责 |
|---------|---------|
| 数据中心管理员 | 机房资源规划与分配 |
| 资产管理员 | 设备资产全生命周期管理 |
| 网络工程师 | 网络资源规划与配置 |
| 采购专员 | 设备采购与供应商管理 |
| 运维工程师 | 日常运维与故障处理 |
| 财务人员 | 成本核算与预算管理 |
| 管理层 | 决策分析与审批 |

## 3. 模块清单

### 3.1 机房IDC管理模块

#### 3.1.1 模块概述

机房IDC管理模块负责数据中心机房的全生命周期管理，包括机房基础信息管理、机柜位管理、空间容量规划等功能。

#### 3.1.2 功能清单

| 功能子模块 | 功能点 | 功能描述 |
|-----------|--------|---------|
| **机房基础管理** | 机房信息管理 | 维护机房基本信息、位置、等级(T1-T4)等 |
| | 机房区域管理 | 划分机房功能区域(主机区、网络区、电力区等) |
| | 容量规划 | 机柜位容量、电力容量、网络容量的规划 |
| **机柜管理** | 机柜信息管理 | 机柜规格、U位容量、承重能力等 |
| | 机柜位视图 | 可视化展示机柜占用情况 |
| | 机柜资源分配 | 机柜位分配与变更管理 |
| **环境管理** | 温湿度监控 | 集成环境监控系统，展示温湿度数据 |
| | 门禁管理 | 机房出入权限管理 |
| | 安防监控 | 视频监控集成与管理 |
| **数据中心可视化** | 3D可视化 | 机房三维可视化展示 |
| | 资产拓扑 | 设备间连接关系拓扑图 |
| | 热力图 | 机房温度、能耗热力分布图 |

#### 3.1.3 核心业务流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  机房规划   │───►│  机柜部署   │───►│  资源分配   │───►│  容量监控   │
```

## 7. 用户认证与管理模块

### 7.1 模块概述

用户认证与管理模块提供系统的身份认证和用户管理功能，基于 Session 认证机制，支持用户登录、登出、用户信息管理等功能。

### 7.2 功能清单

| 功能子模块 | 功能点 | 功能描述 |
|-----------|--------|---------|
| **认证管理** | 用户登录 | 用户名密码登录，Session认证 |
| | 用户登出 | 清除Session，退出登录 |
| | 登录状态检查 | 检查当前用户是否已登录 |
| | 当前用户信息 | 获取当前登录用户详细信息 |
| **用户管理** | 用户列表 | 获取所有用户列表（仅管理员） |
| | 创建用户 | 创建新用户（仅管理员） |
| | 更新用户 | 更新用户角色、状态（仅管理员） |
| | 删除用户 | 删除用户（仅管理员） |
| **密码管理** | 修改密码 | 用户修改自己的密码 |
| **角色管理** | 角色定义 | ADMIN（管理员）、USER（普通用户） |
| | 权限控制 | API访问权限基于角色控制 |

### 7.3 API端点

| 端点 | 方法 | 描述 | 权限 |
|------|------|------|------|
| `/api/v1/auth/login` | POST | 用户登录 | 公开 |
| `/api/v1/auth/logout` | POST | 用户登出 | 登录用户 |
| `/api/v1/auth/current` | GET | 获取当前用户 | 登录用户 |
| `/api/v1/auth/status` | GET | 登录状态检查 | 公开 |
| `/api/v1/users` | GET | 用户列表 | ADMIN |
| `/api/v1/users` | POST | 创建用户 | ADMIN |
| `/api/v1/users/{id}` | GET | 获取用户详情 | ADMIN |
| `/api/v1/users/{id}` | PUT | 更新用户 | ADMIN |
| `/api/v1/users/{id}` | DELETE | 删除用户 | ADMIN |
| `/api/v1/users/me/password` | POST | 修改密码 | 登录用户 |
| `/api/v1/users/me/is-admin` | GET | 检查是否管理员 | 登录用户 |

### 7.4 默认账户配置

系统启动时自动初始化默认管理员账户：
- 默认用户名: `admin`
- 默认密码: `123456`
- 默认角色: `ADMIN`
- 默认状态: `ACTIVE`

可通过环境变量自定义：
- `APP_DEFAULT_USERNAME`: 自定义管理员用户名
- `APP_DEFAULT_PASSWORD`: 自定义管理员密码

### 7.5 安全机制

| 安全措施 | 描述 |
|---------|------|
| 密码加密 | BCrypt算法加密存储 |
| Session认证 | 用户登录信息存储在Session |
| 认证拦截器 | 拦截未登录的API请求 |
| 角色权限 | 基于ADMIN/USER角色的权限控制 |
| CSRF禁用 | 前后端分离场景禁用CSRF |

### 7.6 技术实现

| 技术组件 | 说明 |
|---------|------|
| Spring Security | 禁用默认认证，使用自定义Session |
| BCryptPasswordEncoder | 密码加密组件 |
| AuthInterceptor | 自定义认证拦截器 |
| UserContext | 用户上下文工具类（Session操作） |
| DataInitializer | 应用启动时初始化默认用户 |
